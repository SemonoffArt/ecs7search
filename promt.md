# Промт для создания веб-приложения ecs7search Web UI

## Задача

Создать простое веб-приложение на Python для поиска и визуализации тегов на экранах SCADA ECS7.

Пользователь вводит имя тега (например, `020MG021M01`) в поле ввода. Приложение ищет этот тег в существующем индексе и отображает:
1. Список экранов (файлов `.g`), где упоминается тег.
2. Изображения с жёлтыми рамками, отмечающими позиции тега на скриншотах экранов.

## Технологии

- **Язык**: Python 3.10+
- **Фреймворк**: Flask
- **Менеджер пакетов**: UV
- **Зависимости**: Pillow (для обработки изображений)
- **Без баз данных** — все данные берутся из существующего `data/mimics_index.json`


## Архитектура приложения

### Файл `app.py`

Единый файл Flask-приложения с двумя маршрутами:

1. **`GET /`** — главная страница с формой поиска и результатами (если есть запрос).
2. **`POST /search`** — обработка формы поиска, вызов логики поиска, рендеринг результатов.

### Логика поиска

**Не вызывать `mimic_searcher.py` как CLI-скрипт.** Вместо этого импортировать и переиспользовать ключевые функции из `utils/mimic_searcher.py`:

- `load_index(index_path)` — загрузка JSON-индекса.
- `search_tags(index_data, query)` — поиск по точному совпадению или шаблону (`*`, `?`).
- `get_image_for_file(g_file, sources_dir)` — поиск PNG-скриншота по имени .g файла.
- `convert_ecs_coords(x, y, img_size_x, img_size_y)` — конвертация координат ECS (137×77) в пиксели изображения.
- `draw_border_on_image(png_path, positions, output_path)` — отрисовка жёлтых рамок на изображении.

Импортировать функции из `utils/mimic_searcher.py` с помощью `sys.path.insert()` или относительного импорта.

### Шаблон `templates/index.html`

Одна HTML-страница с:

- Полем ввода для тега (input type="text").
- Кнопкой "Найти".
- Блоком результатов (показывается только после поиска):
  - Список найденных файлов экранов (ссылки или просто имена).
  - Галерея изображений с результатами (каждое изображение — `<img src="...">` из `data/temp/`).
- Минимальный CSS для приятного вида (можно использовать Bootstrap CDN).

## Требования к коду

### Принципы

- **KISS** — максимально простое решение без лишних абстракций.
- **MVP-first** — только ввод тега → поиск → отображение результатов.
- **Explicit over implicit** — прямой, понятный код без "магии".
- **Единая ответственность** — `app.py` только маршруты и вызов логики, логика поиска — в `utils/mimic_searcher.py`.

### Безопасность

- Валидация ввода: разрешать только буквы, цифры, `*`, `?`, `_`.
- Не разрешать path traversal при работе с файлами.
- Не использовать `eval()` или `exec()`.

### Обработка ошибок

- Если индекс не найден — показать понятное сообщение.
- Если тег не найден — показать "Ничего не найдено".
- Если PNG для экрана отсутствует — пропустить и показать предупреждение.
- Глобальный обработчик 500 ошибок.

## Пример ожидаемого поведения

1. Пользователь открывает `http://localhost:5000`.
2. Вводит `020MG021M01` и нажимает "Найти".
3. Приложение:
   - Загружает `data/mimics_index.json`.
   - Ищет тег через `search_tags()`.
   - Для каждого найденного файла рисует рамки на PNG через `draw_border_on_image()`.
   - Сохраняет результаты в `data/temp/`.
4. На странице отображается:
   - Список файлов, где найден тег.
   - Изображения с жёлтыми рамками.

## Замечания

- Все сообщения пользователю — на русском языке.
- Приложение должно работать автономно, без внешних сервисов.
- Не изменять существующие файлы `utils/mimic_indexer.py` и `utils/mimic_searcher.py`.
- Индекс `data/mimics_index.json` уже существует и должен быть загружен при старте.

## Решения по уточнениям

1. **Поиск по шаблону** — реализовать поддержку `*` и `?` в веб-интерфейсе (использовать существующую функцию `search_tags()`).
2. **Скачивание изображений** — не нужно, только просмотр на странице.
3. **Пагинация** — не нужна.
4. **Кэширование** — не нужно, генерировать заново при каждом запросе.
