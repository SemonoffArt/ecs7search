# Промпт для создания PDF Indexer — утилиты индексации тегов в PDF документах

## Контекст проекта

Проект **ecs7search** — Python-утилита для индексации и поиска тегов SCADA системы ECS7.

Существующая архитектура:
- `utils/mimic_indexer.py` — индексирует мимик-файлы (`.g`), извлекает теги по regex `\d{3}[A-Za-z]{2,3}\d{2,3}[A-Za-z0-9_]*`
- `utils/mimic_searcher.py` — поиск по индексу и визуализация на скриншотах
- `data/mimics_index.json` — результат индексации мимик-файлов

## Задача

Написать утилиту `utils/pdf_indexer.py`, которая:

1. **Сканирует** все `.pdf` файлы в директории `./data/pdf/` (без рекурсии, только корневой уровень)
2. **Извлекает текст** из каждой страницы каждого PDF документа
3. **Находит теги** ECS7 в тексте по regex-паттерну: `\d{3}[A-Za-z]{2,3}\d{2,3}[A-Za-z0-9_]*`
   - Примеры тегов: `020ML060Y10I01`, `060TN340A01`, `992CD100G31`, `050BL265`
4. **Составляет индекс** — маппинг `тег → список документов и страниц`
5. **Сохраняет** результат в `./data/pdf_index.json`

## Требования к формату выходного файла

Формат должен быть аналогичен `mimics_index.json`:

```json
{
  "metadata": {
    "directory": "C:\\Users\\...\\data\\pdf",
    "indexed_at": "Sat Apr 12 20:00:00 2026",
    "total_files": 31,
    "total_tags": 1250,
    "total_occurrences": 4800,
    "indexing_time_sec": 12.34
  },
  "tags": {
    "020ML060Y10I01": {
      "files": ["Конвейер 020BC031 REV_6.1.pdf", "Насосная станция 2-го подъёма ЗИФ rev5.pdf"],
      "positions": [
        {"file": "Конвейер 020BC031 REV_6.1.pdf", "page": 3, "count": 2},
        {"file": "Конвейер 020BC031 REV_6.1.pdf", "page": 7, "count": 1},
        {"file": "Насосная станция 2-го подъёма ЗИФ rev5.pdf", "page": 1, "count": 1}
      ]
    }
  }
}
```

### Описание полей

- `metadata` — статистика индексации:
  - `directory` — абсолютный путь к сканируемой папке
  - `indexed_at` — дата/время создания индекса
  - `total_files` — количество обработанных PDF файлов
  - `total_tags` — количество уникальных найденных тегов
  - `total_occurrences` — общее число вхождений тегов (сумма всех count)
  - `indexing_time_sec` — время выполнения в секундах

- `tags` — словарь, ключ = имя тега, значение = объект:
  - `files` — список уникальных имён файлов, где встречается тег
  - `positions` — список вхождений, каждый элемент:
    - `file` — имя PDF файла
    - `page` — номер страницы (1-based)
    - `count` — сколько раз тег встретился на этой странице

## Технические требования

### Библиотеки

Использовать **PyMuPDF** (`fitz`) для извлечения текста из PDF — это быстрая и надёжная библиотека.

```bash
uv add pymupdf
```

Альтернатива: `pdfplumber` или `pypdf`, но PyMuPDF предпочтительнее по скорости.

### Опции командной строки

- `directory` — путь к папке с PDF (по умолчанию `./data/pdf/`)
- `-o, --output` — путь к выходному JSON (по умолчанию `./data/pdf_index.json`)

### Обработка ошибок

- Пропускать повреждённые PDF файлы с предупреждением в stderr
- Игнорировать файлы, не являющиеся `.pdf`
- Обрабатывать PDF с русскими и английскими именами файлов

### Оптимизация

- Использовать `fitz.open()` для каждого файла отдельно (не держать все открытыми)
- Собирать теги постранично, затем агрегировать
- Сортировать итоговый словарь тегов по алфавиту

### Комментарии

- Все комментарии в коде должны быть на **русском языке** (по конвенции проекта)
- Docstrings на русском

## Примеры тегов ECS7 для тестирования

Найди в реальных PDF из `./data/pdf/` (31 файл):
- `020CR020` — в "Дробилка 020CR020 REV_2.pdf"
- `060TN340` — во множестве файлов
- `991CS100`, `992CS100` — в файлах с ревизиями
- `050BL265`, `050BL266` — в "Воздуходувки флотации"

## Что НЕ нужно делать

- Не нужно извлекать координаты тегов (только страница и количество)
- Не нужно создавать поисковый интерфейс (только индексация)
- Не нужно обрабатывать PDF в подпапках (только корневой уровень)
- Не нужно дублировать теги на одной странице (агрегировать count)

## Проверка результата

После генерации индекса должна быть возможность:
1. Открыть `pdf_index.json` и увидеть структуру с metadata и tags
2. Найти конкретный тег и увидеть, в каких файлах и на каких страницах он встречается
3. Общее время индексации 31 файла — не более 30 секунд
